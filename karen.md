# Project: Karen Anniversary Website

## 📌 Project Identity
This project is a personal, long-term digital scrapbook website being built incrementally over time (currently ~3–4 months into the relationship) with the goal of presenting a polished, emotional, and meaningful experience at the one-year anniversary.

The site is NOT a generic blog, NOT a portfolio, and NOT a commercial product.

It is:
- A private, sentimental project
- A living archive of memories
- A curated emotional experience
- A slow-build project (content added continuously over months)

Primary audience:
- Karen (girlfriend)
Secondary audience (optional):
- Close friends/family (if shared)

---

## 🎯 Core Objective

Create a beautiful, minimal, and emotionally impactful website that:
- Documents the relationship over time
- Preserves memories (logs, photos, milestones)
- Feels intentional and personal—not generic
- Evolves gradually and is revealed at the 1-year mark

---

## 🧩 Special Experience Layer — Scavenger Hunt

This website will also function as an interactive scavenger hunt for Karen.

### Intent
- Transform the site into an experience, not just a collection of memories
- Encourage exploration, curiosity, and emotional discovery
- Reward attention to detail and shared inside jokes

### Implementation Ideas
- Hidden clickable elements (e.g., subtle icons, hearts, images)
- Clues embedded in logs or captions
- Timeline entries that unlock future hints
- Passwords derived from shared memories (e.g., first date location)
- Sequential discovery (finding one clue reveals the next)
- “Easter eggs” like hover effects, hidden text, or keyboard triggers

### Experience Goals
- Make Karen actively engage with the site
- Create moments of surprise and delight
- Build emotional payoff as she uncovers deeper layers
- End with a meaningful “final reveal” or message

### Constraints
- Keep puzzles intuitive (not frustrating)
- Ensure everything is solvable using shared experiences
- Maintain emotional tone (never feel like a generic game)

---

## 🧠 Context for AI / ChatGPT

When working on this project, assume:
- The user is continuing an ongoing build, not starting from scratch
- Content will be added over time (not all at once)
- Emotional tone matters as much as technical correctness
- Simplicity > overengineering
- Personalization > scalability

Avoid:
- Overly corporate or product-style suggestions
- Complex backend systems unless explicitly requested
- Generic romantic clichés without specificity

Encourage:
- Specific, vivid, memory-based writing
- Clean UI/UX decisions
- Incremental improvements
- Thoughtful features over flashy ones

## 🏗️ Overall Scope

### Included Features (Core)
- Homepage (hero, intro message, featured content)
- Logs (journal-style entries / letters)
- Timeline (relationship milestones)
- Gallery (photos organized into albums)
- About section (relationship summary)

### Optional Features (Later)
- Playlist/music integration
- Map of locations visited
- “Open later” messages (time-locked)
- Hidden interactions / easter eggs
- Password protection (privacy)
- Scavenger hunt progression system

### Explicitly Out of Scope (for now)
- Authentication systems
- Databases
- User accounts
- Public CMS
- Monetization
